## 🚀 快速配置指南

> 💡 只需 **3 步** 即可跑起来！

### 第1步：配置SMTP邮箱

> ⚠️ **注意：使用QQ邮箱请填授权码，不是QQ密码！**

以 **QQ邮箱** 为例：

```
登录 QQ邮箱 → 设置 → 账户
    ↓
开启「POP3/SMTP服务」
    ↓
生成授权码（16位字母）
    ↓
填入插件配置
```

最小配置示例：

```json
{
  "smtp_host": "smtp.qq.com",
  "smtp_port": 465,
  "smtp_user": "123456789@qq.com",
  "smtp_password": "你的16位授权码",
  "smtp_encryption": "ssl"
}
```

### 第2步：配置验证群

```json
{
  "enabled_groups": ["123456789", "987654321"]
}
```

> 💡 留空 `[]` 则所有群都生效

### 第3步：添加管理员

```json
{
  "admin_qqs": ["你的QQ号"]
}
```

保存配置 → 重启 AstrBot → ✅ 搞定！

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `smtp_host` | string | `smtp.qq.com` | SMTP服务器地址 |

## 🎯 三种验证模式详解

### 模式0：用户自由选择

> `verification_mode = 0` **（推荐）**

用户入群后 @机器人 → 收到选择菜单（回复 `1` 邮箱验证 / `2` 数学题验证）→ 自由选择验证方式

**适合场景**：无邮箱的用户可选数学题，有邮箱的选邮箱验证，灵活包容。

### 模式1：仅邮箱验证

> `verification_mode = 1`

用户入群后 @机器人 → 自动发送验证码到 `QQ号@qq.com` → 群内回复验证码通过

**适合场景**：群内成员都有QQ邮箱，需要更高安全性的验证。

### 模式2：仅数学题验证

> `verification_mode = 2`

用户入群后 @机器人 → 自动出算术题（如 `35 + 28 = ?`）→ 答对通过

**适合场景**：不需要收集邮箱信息，快速简单验证。

## 🔄 用户验证流程

### 📧 邮箱验证流程（模式1）

```
① 用户申请加群 → 自动通过（或手动审批）
        ↓
② 用户 @机器人 回复任意消息
        ↓
③ 机器人发送验证码至 用户QQ号@qq.com
   （同时群内以图片展示验证码，方便对照）
        ↓
④ 用户查收邮件 / 看群内验证码图片
        ↓
⑤ 用户在群内 @机器人 回复6位验证码
        ↓
⑥ 验证成功 ✅ → 机器人发送欢迎消息
```

### 🧮 数学题验证流程（模式2）

```
① 用户申请加群 → 自动通过
        ↓
② 机器人自动出算术题（如 "42 - 17 = ?"）
        ↓
③ 用户 @机器人 回复答案（如 "25"）
        ↓
④ ✅ 答对 → 验证通过
   ❌ 答错 → 重新出题
```

## 📝 变量模板系统

所有消息模板均支持以下变量自动替换，可完全自定义文案。

### 可用变量总表

| 变量 | 说明 | 可用范围 |
|------|------|----------|
| `{at_user}` | @用户 | 群内消息模板 |
| `{member_name}` | 用户昵称/群名片 | 全部模板 |
| `{group_name}` | 群名称 | 全部模板 |
| `{group_member_count}` | 群成员总数 | 全部模板 |
| `{admin_list}` | 管理员列表（"、"连接） | 全部模板 |
| `{email}` | 用户邮箱地址 | 邮件相关模板 |
| `{code}` | 验证码 | 邮件相关模板 |
| `{timeout}` | 超时分钟数 | 全部模板 |
| `{countdown}` | 踢出倒计时秒数 | 超时相关模板 |
| `{bg_url}` | 背景图链接 | 邮件HTML模板专用 |
| `{card_style}` | 毛玻璃卡片CSS样式 | 邮件HTML模板专用 |

### 可自定义模板一览

| 模板名 | 默认内容 | 说明 |
|--------|---------|------|
| `trigger_prompt` | `{at_user} 欢迎加入本群！\n本群当前共 {group_member_count} 位群友\n管理员列表：\n{admin_list}\n请 @我 并回复任意消息...` | 入群提示 |
| `mode_0_menu_prompt` | `{at_user} 欢迎加入！请回复数字选择验证方式：\n1 - 邮箱验证\n2 - 数学题验证` | 模式0选择菜单 |
| `email_sent_prompt` | `{at_user} 验证码已发送至 {email}\n请查看邮件并回复验证码。` | 验证码已发送 |
| `wrong_code_prompt` | `{at_user} 验证码错误，新验证码已发送至 {email}` | 验证码错误 |
| `welcome_message` | `{at_user} 验证成功，欢迎你的加入！🎉` | 验证通过欢迎语 |
| `countdown_warning_prompt` | `{at_user} 验证即将超时，请尽快验证！` | 超时警告 |
| `failure_message` | `{at_user} 验证超时，将在 {countdown} 秒后被请出。` | 超时最后通牒 |
| `kick_message` | `{at_user} 因未完成验证，已被请出本群。` | 踢出公告 |
| `email_subject` | `{group_name} 入群验证码` | 邮件主题 |
| `email_body_html` | （完整HTML模板，见下方） | 邮件正文HTML |

### 邮件HTML模板示例

默认的邮件HTML：

```html
<h2>欢迎加入 {group_name}</h2>
<p>您好 {member_name}：</p>
<p>您的验证码为：<b style='color:blue'>{code}</b></p>
<p>请在 {timeout} 分钟内在群内 @我 并回复此验证码。</p>
```

开启背景图后可用的增强变量：

```html
<div style="{card_style}">
  <h2>欢迎加入 {group_name}</h2>
  <p>验证码：<b>{code}</b></p>
</div>
<img src="{bg_url}" style="opacity:0.3; position:absolute; top:0; left:0; width:100%;">
```

## 🖼️ 视觉特性说明

### 验证码图片

- **渐变色背景**：每次生成不同配色方案
- **噪点与干扰线**：防OCR识别
- **大号数字**：清晰易读
- **在群内以图片形式展示**，方便不方便查看邮件的用户

### 邮件模板选择

插件内置了5种精美的邮件模板供你选择：

| 模板编号 | 名称 | 说明 |
|---------|------|------|
| `1` | 经典蓝色 | 蓝色主题，带毛玻璃效果和背景图支持（默认） |
| `2` | 简约风格 | 简洁清爽，适合正式场合 |
| `3` | 渐变紫色 | 紫色渐变背景，时尚大气 |
| `4` | 卡片风格 | 卡片式设计，优雅稳重 |
| `5` | 科技风格 | 深色科技风，适合技术群 |
| `0` | 自定义模板 | 自由编辑HTML，完全定制 |

在配置中设置 `email_template_choice` 即可切换模板。

### 邮件美化

- **HTML模板**：支持自由编辑HTML
- **二次元背景图**：可开启自动从API获取二次元图片作为邮件背景（仅模板1支持）
- **毛玻璃卡片效果**：验证码区域毛玻璃样式的CSS（仅模板1支持）
- **响应式设计**：所有模板均支持手机端和PC端正常显示

### 欢迎图片

- 支持本地图片路径或网络链接
- 验证通过后随欢迎消息一起发送

## 💡 常见问题

### ❓ 为什么邮件发送失败？

1. **检查授权码**：QQ邮箱必须使用**授权码**，不是QQ密码
2. **检查SMTP设置**：确认 `smtp_host`、`smtp_port`、`smtp_encryption` 正确
3. **检查邮箱是否开启SMTP**：登录邮箱设置确认POP3/SMTP已开启
4. **查看日志**：使用 `新人进群测试日志` 指令查看详细错误

### ❓ 验证码收不到？

1. 检查垃圾邮件箱
2. 确认QQ号正确，邮箱地址为 `你的QQ号@qq.com`
3. 管理员可使用 `发送邮箱测试 QQ号` 指令手动测试

### ❓ 不想开启邮箱验证怎么办？

设置 `verification_mode = 2`（纯数学题验证），无需配置SMTP。

### ❓ 只想在部分群开启验证？

在 `enabled_groups` 中列出指定的群号，留空则全部群生效。

### ❓ 如何自定义提示语？

修改对应的模板变量即可，支持所有 `{}` 变量替换，灵活定制。
